1. Layout fluído

Faz uso de medidas relativas (%, em, rem) para proporcionar fluidez ao site quando o tamanho de tela ou fonte são alterados

PX

Medidas imutáveis, como background ou text-shadow

EM

Valor relativo ao font-size do elemento pai

Utilizado em fontes, margins, paddings

REM

Valor relativo ao font-size do root do documento

Não funciona em algumas versões de Android e IE<9

%

Utilizado em containers, grids fluídos e imagens

CSS Box Model

Conteúdo: caixa onde o conteúdo é exibido

CSS Box Model

Padding: espaçamento em torno do conteúdo

CSS Box Model

Border: a borda que vai ao redor do elemento

CSS Box Model

Margin: espaçamento ao redor da borda do elemento

box-sizing: content-box

O default do browser é o content-box, que considera o tamanho original somado aos espaçamentos (padding, margin, etc).

box-sizing: border-box

A propriedade border-box faz com que o tamanho dessas propriedades não seja somada à largura definida para o elemento.

Aplicação


*, *:before, *:after {
  -webkit-box-sizing: border-box;
     -moz-box-sizing: border-box;
          box-sizing: border-box;
}

Tipografia fluída - EM


body { font-size: 16px; }

h1 { font-size: 24px; }

h1 span { font-size: 18px; }
_____________________________

/* 24 ÷ 16 = 1.5 */
h1 { font-size: 1.5em; }

/* 18 ÷ 24 = 0.75 */
h1 span { font-size: 0.75em; }

Tipografia fluída - REM


body { font-size: 16px; }

h1 { font-size: 24px; }

h1 span { font-size: 18px; }
_____________________________

h1 { font-size: 1.5rem; }
/* 24 ÷ 16 = 1.5 */

h1 span { font-size: 1.125rem; }
/* 18 ÷ 16 = 1.125 */

Tipografia fluída - um facilitador....

Para evitar perda de tempo e resultados quebrados, podemos atribuir ao font-size do html o valor de 62.5%. Assim sendo, o valor de 1EM equivalerá a 10px, facilitando os cálculos!


htm1 { font-size: 62,5%; }

h1 { font-size: 15px; }

h1 span { font-size: 11px; }
_____________________________

htm1 { font-size: 62,5%; }

h1 { font-size: 1.5em; }

h1 span { font-size: 1.1em; }

2. Layout adaptável

Consiste em modificar, e/ou disponibilizar, ou não, determinados elementos de acordo com o tamanho de tela, adaptanto o conteúdo mais relevante ao espaço disponível.

A viewport

<meta name="viewport" content="width=device-width, initial-scale=1.0"/>


Viewport é a aréa de visualização do dispositivo, onde o site é exibido.


A customização do viewport serve para deﬁnir a resolução inicial de visualização do site, e evitar a miniaturização do mesmo.

Break Points

Pontos de mudanças consideráveis no layout da página.

Exemplos

Menu

Disposição de elementos

Inibição de conteúdo

Media Queries

Expressões condicionais para aplicar diferentes regras de CSS, dependendo da largura do viewport e/ou orientação da tela, por exemplo.

@media screen and (max-width:480px)
(resolução máxima no viewport de 480px)

@media all and (orientation:landscape)
Orientação “paisagem”

@media screen and (min-width: 400px) and (max-width: 700px)
Tela com resolução mínima de 400px e máxima de 700px

3. Conteúdo flexível

A mesma fluidez e adaptação de elementos estruturais se aplica aos componentes de mídia.

Imagens responsivas

Cálculos matemáticos de um ponto ao outro que constroem formas geométricas.

Imagens feitas de pixels. Um ponto único e mínimo na tela de um dispositivo.

SVG

SVG é a abreviatura de Scalable Vector Graphics que pode ser traduzido do inglês como gráﬁcos vetoriais escaláveis.

Trata-se de uma linguagem XML para descrever de forma vetorial desenhos e gráﬁcos bidimensionais, quer de forma estática, quer dinâmica ou animada.

Incorporamos nos sites de forma responsiva utilizando a tag object, para poder fornecer uma opção alternativa de imagem a navegadores que não suportam SVG.

Mobile First

O desafio

O excesso de informação em grande parte dos sites se dá porque é relativamente fácil adicionar conteúdo na versão desktop.

A proposta

￼Desenhe sua solução primeiro para mobile, depois para desktop.

Como?

Estabelecer uma escala hierárquica de importância das informações textuais e gráficas do site

Repensar a pertinência de apresentação dessas informações em diferentes contextos e dispositivos

Steve Souders, desenvolvedor do Google e autor do livro "High Performance Web Sites"
